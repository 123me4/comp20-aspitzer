Security Assessment

By Alex Spitzer

Introduction

This assignment was to find as many security and privacy problems in the jordan-marsh heroku app (aka Not Foursquare) as possible.
The app allows users to log in with a username and location (in the form of longitude and latitude). They can see when and where others have logged in as well on the main page.

Methodology

At first I did some black box testing, by playing around with query strings on the login and main page. After that I reviewed the sourcecode and attempted some white box testing, using more query strings and curl.

Abstract of Findings

Not Foursquare allows users to provide unexpected input, which allows for the threat of malicious hacking.

Issues Found

  • Cross-site scripting

    • Location - GET pages
    • Severity - high.
    • Description - This issue allows users to insert scripts into the site when the site loads their login. A user could force a pop up, insert any image they wish, or even redirect the entire page to another site.
    • Resolution - Sanitize the "login" variable. Whitelist characters like '<', '>', ';', '}', etc.

  • NoSQL Injection

    • Location - GET pages
    • Severity - high.
    • Description - This issue allows users to find and receive information they shouldn't have. Adding the query string "?login[$gt]=" to /checkins.json provides the entire JSON of the database, with all the data connected to every login.
    • Resolution - don't let the query be completely user input.

Conclusion

The app as it is currently coded risks hackers harming both users in their own browser, and data in the server. It needs to be fixed to prevent as much damage as possible.
